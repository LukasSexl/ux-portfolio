01At-a-Glance
The goal was to explore a UX Law. I chose Parkinson’s Law, which states: “Work expands so as to fill the time available for its completion.” So I designed a generic banking app with a task for the user to complete in a certain amount of time.

Hypothesis: If I provide more time for the same task, more time will be used to complete it.

Then I did A/B Testing with the time as a variable. Spoiler: I was able to prove said law.

03Prototyping Process
I thought that it needed to be something everyone can relate to, but be complex enough so that enough time can be spent.
I decided to replicate the user flow in a banking app of transferring 50€ to one’s grandma.



04A/B Testing
Having completed the Prototype, I used Maze to upload and create two different tests.
The reason for choosing this tool is that it accurately measures the time and much more other interesting data.

Test A was with 15 seconds, Test B with 30 seconds.
The goal was to to find out if the users will use up the given amount of time.

I sent the links to 5 users each and while testing I held a stopwatch next to their screen, so they had a better feeling for the time.
(I tried doing a stopwatch in my Figma prototype, but unfortunately it started playing before the user pressed on “begin” in Maze).
05Results
15 seconds: 10.4 seconds (Report)
30 seconds: 18 seconds (Report)

The results kind of proved the Hypothesis!

One could argue that people didn’t use up all the 15 seconds or 30 seconds.
On the other hand, it is understandable that the amount of time spent is less than the given amount of time since the users wanted to finish “in time” (earlier than the deadline).

But 7.6 seconds difference in average duration is fairly significant.
06Iterations
To increase the time spent in the prototype by just a bit, I changed the selection process of the amount:The user will have to type 5, 0, 0, 0 to get to the 50€. I intend that this will be more complicated → increased thinking → more time.
07What I learned
I learned how to use Maze, a tool which I like and that I’m sure will prove itself to be useful in the future due to its comprehensive data analytics. Also, I refined my design skills in Figma and it was interesting to see the approaches of my colleagues to other laws.
